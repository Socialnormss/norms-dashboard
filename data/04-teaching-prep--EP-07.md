# NPC Gen2 — EP 7 · POI (Point of Interest) — แยก 3 ตัว + Demand/Supply คือ Liquidity

> Lesson note by Norms (Max) · 2026-05-21 · sources: Turbo transcript

## TL;DR
- **POI = Point of Interest** = จุดรวมความสนใจของคน (ไม่ใช่ของเราคนเดียว) — แปลตรงตัว = "พื้นที่ที่คนสนใจจะเทรด"
- POI แบ่งเป็น 3 ตัวหลัก: **PY Lick** (สีฟ้า · นอก Flow · ตัวแรกที่เห็นชัด) · **PY ช่วยๆ** (สีส้ม · ตัวที่ 2) · **PY Focus** (สีแดง · อยู่ใต้สองตัวบน · ในโซน Flow)
- **Demand/Supply Zone = Liquidity ทั้งหมด** — ไม่ใช่ "พื้นที่ซื้อ/ขาย" จริง · มีไว้ให้คนแห่เข้ามาแล้วโดนกิน · เราจึงไม่เทรดตามมัน แต่รอให้มันโดน Clear ก่อน
- หลัก Time Frame: 1H = กำหนดข้อมูล/Trend · 15M = แผน · 5M = Confirm · สลับ HTF/MTF/LTF ได้ตามสถานการณ์
- **กฎปลอดภัย:** อะไรที่อยู่ **นอก Flow** ไม่เทรด · อะไรที่อยู่ **ใน Flow** ให้นัยสำคัญสูง · ทุกเงื่อนไขต้องผ่านเส้นเวลา (Time)

## 🎯 หัวข้อใน EP นี้
1. Time Frame Management — เมื่อไหร่ Fix 1H · เมื่อไหร่ใช้ 15M เป็นโครงสร้าง
2. POI คืออะไร · 3 ตัวแยกยังไง (PY Lick / PY ช่วยๆ / PY Focus)
3. Demand/Supply Zone (Rally-Base-Rally / Drop-Base-Drop) — ทำไม "มันคือ Liquidity"
4. แนวรับแนวต้าน vs การสะสม — อะไรเกิดก่อน
5. Multiple Time Frame: เส้น → จุด → โซน → Order Flow (เรื่องเดียวกัน คนละ TF)
6. กฎ "นอก Flow ไม่เทรด · ใน Flow ใช้ได้"
7. การหดโซน (Reduce Zone) เพื่อ Entry ที่ดี + ลด SL
8. Flip Zone (= Baker ของชาวบ้าน) — preview ก่อนเข้า EP ถัดไป

## 📚 เนื้อหาเต็มที่หลิวสอน

### 1) Time Frame Management — 1H / 15M / 5M

หลิวย้ำตั้งแต่ต้น EP ว่าในอุดมคติ **ต้อง Fix Time Frame** เป็น 1H / 15M / 5M

- **1H** = ไว้ครอบทั้งหมด · กำหนด **Trend + Momentum** + ใส่ข้อมูลในยะสำคัญ
- **15M** = ใช้ **วางแผน**
- **5M** = ใช้ **Confirm**

แต่ความจริงในหน้างานคือ "บางวัน 1H วางแผนเทรดไม่ได้" — แท่ง 1H ไม่ขยัก / Equilibrium ตลอด / ไม่มีเนื้อเทียนอ่านได้

**วิธีคิดเมื่อ 1H ใช้ไม่ได้:**
- ขยับโครงสร้างลงมาเป็น 15M เป็นหลัก · แล้ว 5M เป็นแผน · 1M เป็น Confirm
- หรือใช้ 1H กำหนด Direction (Buy/Sell · ขึ้น/ลง · Momentum) ทิ้งไว้ · ถ้า 1H ไม่มีโซนชัด → ข้ามไปใช้ 15M หาโซนเอง
- **หลักคิด:** "ยังไงผมก็ต้องมี 1H คุมไว้อยู่ดี ไม่ให้พลาด"

> ตัวอย่างหลิวเล่า: ถ้า 1H เป็นขาขึ้น Break Out + Retest เพื่อขึ้น → เราวางแผน Buy · ถ้าใกล้เวลาข่าวแล้วราคามาไม่ถึงโซน 1H → ปรับมาดู 15M ว่ามีข้อมูลให้เล่นไหม

**เหตุผลที่หลิว "ไม่ค่อยอยากให้ใช้ TF เล็ก":** มันเจอชุดข้อมูลให้เล่นแบบยับยับ → เหนื่อย · เห็นทุก Flow · ขยักทุกขยัก · ตัดสินใจไม่ทัน

### 2) POI (Point of Interest) — แก่นของ EP นี้

#### นิยาม

**POI = Point of Interest** = "จุดรวมความสนใจ" — ใครสนใจ? **คนอื่นสนใจ** ไม่ใช่เรา

> "POI คือจุดที่มีการซื้อขายเกิดขึ้น = เทียบเท่ากับเป็น Liquidity"

หลิวประกาศกฎเหล็ก: **"ต่อไปนี้ POI โดยส่วนใหญ่ผมจะตีว่าเป็น Liquidity ทั้งหมด"** — ไม่ว่าจะเรียก Demand / Supply / Order Block / Key Level / Fibonacci / Indicator → ทั้งหมด = Liquidity

ทำไมต้องคิดแบบนี้? เพราะเป้าหมายของเราในวันๆ นึงคือ:
- ให้คนอื่นเข้ามาเทรด "ผิดที่ผิดเวลา"
- ผิดที่ = เทรดก่อนที่มันควรจะเป็น
- ผิดเวลา = เทรดก่อนเวลาที่ควรจะเป็น
- → เราจะให้สิ่งนั้นเป็น "ก้อนเงิน" ทั้งหมด

#### POI 3 ตัวที่หลิวแยก

| ตัว | สี | ตำแหน่ง | หน้าที่ |
|---|---|---|---|
| **PY Lick** | ฟ้า | นอก Flow · ตัวแรกที่เห็นชัดสุด | สะสมเพื่อสร้าง Liquidity แล้วโดนทำลาย (Sweep) |
| **PY ช่วยๆ** | ส้ม | นอก Flow · ตัวที่ 2 | ตัวเสริม · คนยังเห็น/ยังสนใจอยู่ |
| **PY Focus** | แดง | **ใน Flow** · ใต้สองตัวบน | จุดที่เรา Focus เข้าเทรดจริง |

> **กฎจำง่าย:** "อะไรก็ตามที่อยู่นอก Flow → ตีเป็น Liquidity ไปหมด · อะไรอยู่ในโฟ → ให้นัยสำคัญเยอะหน่อย"

#### ทำไม "ตัวแรกสุด" ส่วนใหญ่ไม่หลุด

หลิวให้สถิติคร่าวๆ จากประสบการณ์: **"ตัวแรกสบกระตุก 100% · หลุดไปเลยไม่เกิน 5%"**

เหตุผล: เมื่อราคา Break Out รุนแรง → คนตกใจ FOMO Buy → ราคา Retest ครั้งแรก → คนที่ FOMO มารีบปิดเอากำไรเล็กๆ · คนที่ Sell ก็รีบปิดกลัวขาดทุน → **เกิด "ปฏิกริยา" (Action) เสมอ**

> "ตัวแรกผมเห็น ไม่ต้องแต่เทรด — เห็นน้อยครั้งที่มันผ่านพรวดลงไปเลย · ตัวแรกสบกระตุก 100%"

#### PY Focus = จุด Focus จริง

PY Focus อยู่ใต้ Lick + ช่วยๆ → คือพื้นที่ที่ **อยู่ใต้ Liquidity** (ใต้ก้อนที่จะโดนกินเสียก่อน)

เงื่อนไขเข้า Focus:
1. ก้อนข้างบน (Lick + ช่วยๆ) ต้อง **โดน Clear ก่อน** (Sweep)
2. ผ่าน **เส้นเวลา** (Time-based filter) — ถ้าราคามาก่อนเวลา → ยังไม่เล่น

**Mindset หลัก:** "ยิ่งอยู่ต่ำ ยิ่งปลอดภัย · ยิ่งรอไกล ยิ่งได้เห็นมากกว่าคนอื่น"

### 3) Demand/Supply Zone — แท้จริงคือ "Liquidity"

#### หน้าตา 2 แบบ

**Demand Zone (Buy):**
- **Rally-Base-Rally (RBR)** = ขึ้น-พักตัว-ขึ้น → Continued (Continuation)
- **Drop-Base-Rally (DBR)** = ลง-พัก-ขึ้น → Reversal

**Supply Zone (Sell):**
- **Drop-Base-Drop (DBD)** = ลง-พัก-ลง → Continued
- **Rally-Base-Drop (RBD)** = ขึ้น-พัก-ลง → Reversal

#### วิธีตีกล่อง (ที่หลิวสอน)

> "ผมตีจาก **เนื้อเทียนไปไส้เทียน**"

- **Demand:** ตีจาก **เนื้อข้างล่าง → ไส้ข้างบน**
- **Supply:** ตีจาก **เนื้อข้างบน → ไส้ข้างล่าง**

ทำไมถึงตีแบบนี้? ไม่ใช่ตีไส้-ถึง-ไส้?
- เพราะ **ความเป็นจริง** ราคาส่วนใหญ่จะเข้ามาที่เนื้อเทียนแล้วปิดเนื้อ ก่อนค่อยขึ้นไป
- ตีจากเนื้อ → SL สั้นกว่า · จุดเข้าแคบกว่า · ปลอดภัยพอ
- "ทำไมต้องฝืนใจวางครั้งบน 50 จุด แล้วต่อข้างล่าง 250 จุด ในเมื่อเอาแค่ 250 ก็พอ"

#### ทำไมหลิวบอก "Demand/Supply ก็คือ Liquidity"

> "พอคนถูกสอนว่าหน้าตาแบบนี้เป็นพื้นที่ซื้อ → ทุกคนก็แห่เข้ามาซื้อตรงนี้ → **Liquidity มันเลยก่อเกิด** → มันจึงโดน Sweep / Clear ได้ทุกครั้ง"

นี่คือเหตุผลที่หลิวบอกว่า **"FVG / Order Block / Demand / Supply ผมไม่เห็นใครใช้ได้สำเร็จเลย · เงื่อนไขมันง่ายมาก คือคุณต้องแพ้ก่อนถึงจะเทรดได้"**

> "เราอยู่ฝั่งผู้สร้าง · เราต้องอยู่ฝั่ง Market Maker · ไม่ใช่คนที่จอดต่อจิกซอ"

#### Demand/Supply โดนทำลาย → กลายเป็น Flip Zone

ถ้า Demand Zone โดนเบรกลงไป → กลับมาเป็น **Flip Zone** (เคยรับ → กลายเป็นต้าน · เคย Demand → กลายเป็น Supply)

หลิวเรียก **Flip** แทนคำว่า "Baker" ของชาวบ้าน
- เพราะหลิวไม่ได้เข้าใจ Baker ลึกซึ้งพอจะใช้คำนั้น
- Flip ของหลิว = จุดที่มีนัยสำคัญร่วมเยอะๆ + เป็น Zone (ไม่ใช่แค่เส้น)

### 4) แนวรับ/แนวต้าน vs การสะสม — อะไรเกิดก่อน?

หลิวยกประเด็นนี้เพื่อโต้แนวคิด "Demand/Supply Zone เกิดเอง"

> "ผมเทียงขาดใจ: **แนวรับ-แนวต้านต้องมาก่อน** · ไม่งั้นอยู่ดีคนจะเข้าไปซื้อขายตรงนั้นได้ยังไง"

ดังนั้นถ้าเราแม่นเรื่อง:
- **โครงสร้าง** (Structure)
- **Hi/Low** ที่สำคัญ
- **แนวรับ-แนวต้าน** ที่ดี
- **ราคาปิด** ที่ดี

→ Demand / Supply / I-Zone / สะสม → **มาเองในอนาคต**

> "ถ้าคุณเก่ง Structure ปุ๊บ — Demand Supply, I-Zone, สะสม → เดี๋ยวอนาคตมาเอง คุณจะหาเจอเอง"

### 5) Multiple Time Frame — เส้น/จุด/โซน/Flow คือเรื่องเดียวกัน

แนวคิดสำคัญที่หลิวอธิบายซ้ำหลายรอบ:

> **"เส้น" ใน Time Frame ใหญ่ = "โซน Demand/Supply" ใน Time Frame กลาง = "Order Flow" ใน Time Frame เล็ก = "เส้นเดียว" ใน Time Frame ใหญ่กว่า**

ลำดับการขยาย (zoom in/out):
1. **Daily / 4H** เห็น → "**เส้นเดียว**" (แนวรับ/แนวต้าน)
2. **1H** เห็น → "**Demand/Supply Zone**" หรือ "**Order Block**"
3. **15M** เห็น → "**Order Flow + POI**"
4. **5M / 1M** เห็น → "**Market** โครงสร้างขยุกขยุย"

> "เส้นเกิดจากจุด 2 จุด · ใน TF ใหญ่เป็นแค่เส้น · ใน TF เล็กลงไป Zoom เข้าไป → ก็จะเป็น Zone · Zoom เข้าอีก → ก็จะเป็นโครงสร้างขยัก"

**บทเรียนสำคัญ:** ไม่ต้องไปนั่งแข่งกันตี Demand/Supply ให้สวย ให้เป๊ะ — **หา High/Low ที่ดีให้เจอก่อน · ตี แนวรับ/แนวต้าน ให้แม่น · ที่เหลือมันมาเอง**

---

### 6) กฎ "นอก Flow ไม่เทรด"

ย้ำซ้ำตลอด EP:

- **อยู่ใน Order Flow** → ให้นัยสำคัญสูง · เป็นจุดที่เราจะ Focus เข้าเทรด
- **อยู่นอก Order Flow** → ไม่เทรด · ตีเป็น **Liquidity (Lick) ไปเลย** · ไม่ต้องสนใจอะไรเยอะ

> "ชุดติด Flow เราไม่เทรด · ย้ำหลายๆ รอบเลยนะครับ · แต่ถ้าจะเทรดชุดติด Flow → ต้องแถลไปเลย · SL ที่ Flow → ถ้าขยึกขยือ ไม่ต้องรอให้มันไหลมา SL → ตัดไปก่อนเลย"

**ข้อยกเว้น:** ถ้าตัว Flow เป็น **PY Lick** (ตัวที่จะโดน Sweep) → กลายเป็น "เล่นในชุดติดโฟ" → ไม่เทรดอยู่ดี

---

### 7) การหดโซน (Reduce Zone) — เพื่อ Entry ที่ดี

แนวคิด: **กรอบใหญ่ → หด → ให้แคบลง** เพื่อ Entry แคบ + SL สั้น

ขั้นตอน:
1. **1H** กำหนด Order Flow ใหญ่ (ก้อน Demand/Supply 1H อาจกว้าง 1,000 จุด)
2. **15M** หาข้อมูล POI ภายใน Flow 1H นั้น
3. ถ้า 15M มี **Flip Zone** + **Fibo 50%** + **SRF** + **BSL** **ทับซ้อนกัน** → "โซนน่าสนใจมาก"
4. **5M** Confirm Action

วิธีหด:
- ถ้ามี POI ใน 15M ที่ทับซ้อนกับ POI 1H → **หดขึ้น** ไปอยู่ที่จุดทับ
- ไม่หด SL ลง · หด Entry ขึ้น (เพื่อ SL คงที่ปลอดภัย)

> "เรามักจะหด Entry ขึ้น ไม่หด SL ลง — เพื่อลดความเสี่ยง + ป้องกันการตกรถ"

### 8) Flip Zone — preview ก่อนเข้า EP ถัดไป

หลิวบอกชัดว่า **Flip = Baker ของชาวบ้าน** แต่หลิวขอเรียก Flip เพราะเข้าใจมันในแบบของตัวเอง

Flip Zone เกิดจาก:
- Demand/Supply Zone โดนทำลาย (Break)
- กลับมาเป็น Zone ตรงข้าม (Demand → Supply, Supply → Demand)

**เงื่อนไข Flip ที่ดี (preview):**
- มีนัยสำคัญร่วมเยอะ (Hi/Low + Fibo 50% + SRF + แนวรับเดิม)
- อยู่ในโครงสร้างหลัก (อยู่ใน Flow)
- คนเคยเชื่อในจุดนี้มาก่อน → มี Liquidity สะสม

หลิวจะลงรายละเอียด Flip + Combo กับ POI ใน EP 8

## 🌐 ข้อมูลสากลเสริม

### Wyckoff/SMC Theory เชื่อมโยง

**กลไก POI ใน Framework สากล:**

- **Wyckoff Phase B (Accumulation/Distribution):** ก้อน Sideway/Base ที่หลิวเรียก POI = Phase B = ช่วงสะสม Composite Operator กำลังเก็บ position
- **Wyckoff Spring / UTAD:** ตรงกับ "PY Lick โดน Sweep" — ราคาทะลุ Low (Spring) หรือ High (UTAD) เพื่อกิน Liquidity แล้วกลับตัว
- **SMC Liquidity Sweep:** การที่ราคาวิ่งไปกินก้อน BSL/SSL = หลักการ "Sweep ก่อน Reverse" — ตรงกับ Wyckoff Spring/UTAD ในรูปแบบ TF เล็ก
- **SMC Order Flow:** ตรงกับการเบรกโครงสร้าง (BOS) + การหา POI ที่อยู่ใน Flow
- **Wyckoff Cause & Effect:** ระยะเวลาที่ราคาสะสมใน POI ยิ่งนาน → Effect (การวิ่งหลังเบรก) ยิ่งไกล — สอดคล้องกับที่หลิวบอก "Pause นานๆ โอกาส Sweep สูง"

**Concept "Demand/Supply = Liquidity" ในมุมสากล:**
- Composite Operator (Wyckoff) วาง trap ให้ retail trader เห็น "Demand Zone สวยๆ" แล้วเข้าซื้อ → จากนั้น Sweep ลง → กิน Stop ของ retail
- นี่คือเหตุผลที่ Wyckoff สอนให้ดู **Volume Confirmation** ทุกครั้งก่อนเชื่อโซน — เพราะโซนเปล่าๆ ที่ไม่มี Volume Climax ไม่ใช่ Demand จริง

### Trader Psychology

**Mindset ที่หลิวฝัง:**

1. **"คุณต้องเป็น Market Maker · ไม่ใช่ Player"** — หยุดพยายามชนะตลาด · เริ่มคิดว่าตลาดวางแผนยังไงให้คนแพ้
2. **"ปล่อยให้คนอื่นต่อจิกซอ · เราค่อยมาจัดการ"** — รอ Liquidity Build เสร็จก่อน · อย่า front-run
3. **"ตัวแรกสบกระตุก 100%"** — Trust the wick · อย่ารีบเข้า Demand แรกที่เห็น
4. **"ยิ่งรอไกล ยิ่งได้เห็นมาก"** — Patience pays · TP เล็กๆ แต่บ่อยๆ ดีกว่า TP ใหญ่แต่นานๆ ครั้ง
5. **TP Discipline:** "ก่อนจะ TP 1,000 ได้ ต้องผ่าน 100 · 200 · 300 มาก่อน" — ฝึก 100 จุด × 30 ครั้ง = Level Up

**Common Mistake ที่หลิวเตือน:**
- เห็น Demand สวย → กด Buy ทันที (ไม่รอ Sweep)
- เห็น POI ใน 5M → เทรด (ไม่เช็ค 1H ว่าเป็น Flow ทิศไหน)
- ใช้ FVG / Order Block แบบไม่เข้าใจที่มา (จำมาใช้)
- ไม่ดูเวลา (Time) ประกอบ — ราคามาก่อนเวลา = เกมยังไม่จบ

## 📝 การบ้าน

### Practice (ฝึก)

1. **TF Mapping:** เปิดกราฟ XAUUSD ใน TradingView → Fix Time Frame เป็น 1H · 15M · 5M
   - 1H: ตี Order Flow + High/Low หลัก
   - 15M: หา POI 3 ตัว (Lick / ช่วยๆ / Focus) ในแต่ละ Flow
   - 5M: ดู Action ที่ POI Focus

2. **Color Coding:** ตี POI ด้วยสีตามที่หลิวสอน
   - **ฟ้า** = PY Lick (นอก Flow · ตัวแรก)
   - **ส้ม** = PY ช่วยๆ (นอก Flow · ตัวที่ 2)
   - **แดง** = PY Focus (ใน Flow · จุดเข้าจริง)

3. **Demand/Supply Drill:** หาตัวอย่างใน chart
   - Rally-Base-Rally (RBR) → Continued Demand
   - Drop-Base-Rally (DBR) → Reversal Demand
   - Drop-Base-Drop (DBD) → Continued Supply
   - Rally-Base-Drop (RBD) → Reversal Supply
   - ตี **จากเนื้อเทียนไปไส้** (ไม่ตีจากไส้ถึงไส้)

4. **Multi-TF Test:** เลือก 1 จุดในกราฟ
   - ดูใน Daily → เห็นเป็น "เส้นเดียว"
   - ดูใน 1H → เห็นเป็น "Demand/Supply Zone"
   - ดูใน 15M → เห็นเป็น "POI + Order Flow"
   - ดูใน 5M → เห็นเป็น "โครงสร้างขยัก"
   - → พิสูจน์ว่ามันคือ "เรื่องเดียวกันคนละ TF"

5. **Sweep Test:** สถิติส่วนตัว — เปิด 1H ไล่ดู 20 ครั้งล่าสุดที่ราคาเจอ POI ตัวแรก
   - กี่ครั้งที่ราคา Sweep ก่อน Reverse?
   - กี่ครั้งที่ราคา Pass-through เลย?
   - เทียบกับสถิติของหลิว (Sweep 95%+ · Pass-through <5%)

### Reflection (สะท้อนคิด)

- คุณเคยเทรด Demand/Supply Zone โดยไม่รอ Sweep หรือไม่? ผลเป็นยังไง?
- คุณเข้าใจคำว่า "นอก Flow ไม่เทรด" ตรงกับที่หลิวสอนไหม? ลองอธิบายในภาษาตัวเอง
- ถ้าให้คุณอธิบายให้คนที่ไม่เคยเรียน ว่า "Demand Zone จริงๆ คืออะไร" คุณจะตอบยังไง?
- ทำไมหลิวบอกว่า "ตัวแรกสุดที่เห็น = PY Lick" — มันสะท้อน Psychology ของคนทั่วไปยังไง?

## 📖 คำศัพท์ที่ควรรู้

| คำศัพท์ | ความหมาย |
|---|---|
| **POI (Point of Interest)** | จุดรวมความสนใจของคนในตลาด · = Liquidity |
| **PY Lick** | POI ตัวแรกที่เห็นชัด อยู่นอก Flow · มีหน้าที่สะสมแล้วโดน Sweep |
| **PY ช่วยๆ** | POI ตัวที่ 2 อยู่นอก Flow · ตัวเสริม |
| **PY Focus** | POI ที่อยู่ใน Flow · จุดที่เรา Focus เข้าเทรดจริง |
| **Order Flow (Flow)** | ทิศทางหลักของราคาใน TF ที่เลือก (Bullish/Bearish) |
| **Demand Zone** | พื้นที่ที่คนถูกสอนว่าให้ซื้อ (Rally-Base-Rally / Drop-Base-Rally) |
| **Supply Zone** | พื้นที่ที่คนถูกสอนว่าให้ขาย (Drop-Base-Drop / Rally-Base-Drop) |
| **Rally-Base-Rally (RBR)** | ขึ้น-พัก-ขึ้น = Continued Demand |
| **Drop-Base-Rally (DBR)** | ลง-พัก-ขึ้น = Reversal Demand |
| **Drop-Base-Drop (DBD)** | ลง-พัก-ลง = Continued Supply |
| **Rally-Base-Drop (RBD)** | ขึ้น-พัก-ลง = Reversal Supply |
| **Flip Zone** | Demand/Supply ที่โดนทำลาย กลับมาเป็นโซนตรงข้าม (= Baker ของชาวบ้าน) |
| **BSL (Buy-Side Liquidity)** | Liquidity เหนือ High (Stop ของ Seller) |
| **SSL (Sell-Side Liquidity)** | Liquidity ใต้ Low (Stop ของ Buyer) |
| **Sweep / Clear** | ราคา Wipe ผ่านโซน Liquidity แล้วกลับตัว |
| **Key Level** | จุดที่มีนัยสำคัญร่วม (Fibo 50% / SRF / BSL / Flip) |
| **HTF / MTF / LTF** | High / Medium / Low Time Frame (1H / 15M / 5M) |
| **Fix Time Frame** | กำหนด TF ตายตัวเพื่อความสม่ำเสมอในการอ่าน |
| **Equal High / Equal Low** | High หรือ Low ที่เรียงกันแบบเสมอ → มี Liquidity ติด |
| **Order Box** | คำเรียก Demand/Supply Zone อีกชื่อนึง · หลิวไม่ใช้ |
| **Baker** | คำของคนอื่น = Flip Zone ของหลิว |
